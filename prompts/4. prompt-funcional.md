# Prompt: Documentación Funcional – NominaPro

## Contexto del Proyecto
- **Nombre**: NominaPro
- **Frontend**: Vue.js
- **Backend**: Python + FastAPI
- **Base de datos**: SQLite (Desarrollo Local) / PostgreSQL con psycopg[binary] (Producción)
- **Enfoque**: sistema educativo de nómina, simple e incremental

## Objetivo
Generar el archivo `/docs/2. funcional.md` de **NominaPro**.

## Estado actual que debe reflejarse
- El usuario crea, consulta y elimina empleados y novedades con llamadas reales al backend.
- La nómina se liquida contra datos persistidos; no debe existir lógica de simulación en memoria.
- Los errores 400/422 deben mostrarse de forma entendible para el usuario final.

## Instrucciones

Genera un documento funcional en Markdown que incluya:

### 1. Descripción del Sistema
- Qué resuelve NominaPro en su versión actual.
- Usuarios objetivo (administrador/auxiliar de nómina en contexto PYME).
- Alcance funcional actual y propósito pedagógico del sistema.

### 2. Módulos

#### 2.1 Gestión de Empleados
- Crear, listar y eliminar empleados.
- Campos mínimos: nombre, documento, salario base, tipo de contrato.
- Restricción de unicidad por documento.

#### 2.2 Gestión de Novedades
- Registrar, listar y eliminar novedades por empleado y período (`YYYY-MM`).
- Variables soportadas: horas extra, ausencias en días, bonificación, descuento extra.
- Upsert por `(empleado_id, periodo)`.

#### 2.3 Liquidación de Nómina
- Generación mensual para empleados activos.
- Cálculo de devengado, deducciones y neto.
- Consulta de nóminas por período.

### 3. Reglas de Negocio
- Valor hora base = `salario_base / 240`.
- Pago horas extra = `horas_extra * valor_hora * 1.25`.
- Descuento ausencias = `(salario_base / 30) * ausencias_dias`.
- 
- Neto = `devengado - deducciones`.

### 4. Flujo Principal Mensual
Incluye un diagrama Mermaid (`flowchart` o `sequenceDiagram`) con:
- Registro/actualización de novedades.
- Disparo de generación de nómina del período.
- Cálculo empleado por empleado.
- Persistencia de resultados y consulta final.

### 5. Supuestos y Límites de la Versión
- Protección por JWT y roles (RBAC) basados en auditoría (UGPP).
- Sin histórico avanzado ni auditoría completa.
- Cálculo simplificado (no contempla todas las particularidades legales de nómina).

### 6. Funcionalidades Recomendadas (Roadmap)
- Autenticación JWT + roles.
- Edición/estado de nómina (borrador, cerrada).
- Exportación de desprendibles (PDF/Excel).
- Reportes y trazabilidad de cambios.

## Formato de Salida
- Markdown bien estructurado con headers, listas, tablas y diagrama Mermaid cuando aplique.
